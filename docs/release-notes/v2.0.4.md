## v2.0.4

> Reworks model variants into independent, custom-named entries (a variant can now be its own model id with a forced behavior), adds one-click transform rule templates, and surfaces cache token counts in the usage explorer.

### English

#### Added

- **Named model variants.** Model variants now carry a custom, absolute name independent of the base model id — a variant of `gpt-5.5` can be named `gpt-image-2` (or anything) and pin a forced behavior such as the image-generation tool, a reasoning level, or a service tier. The provider model editor gained a structured variant editor; each variant's behavior compiles into a rewrite rule keyed by the variant's full name, and the variant resolves to its base model upstream.
- **Transform rule templates.** The rule editor gained one-click transform templates for common client-identity normalizations (Aider, Cline, Continue, Cursor, OpenCode), plus richer transform configuration — multi-path text replacement with explicit `from` / `with` pairs.
- **Cache token columns in usage.** The usage table and chart now surface cache-write and cache-read token counts alongside input / output, with matching rollup and persistence support across the file, SQLite, and libSQL backends.

#### Changed

- **Variant config format (breaking, no back-compat).** `variants_json` now stores absolute variant names — `{ "expose_base": true, "variants": ["gpt-image-2"] }` or a bare name array — and the old suffix forms (`["-thinking"]` / `{ "suffixes": [...] }`) are no longer parsed. Existing suffix-based variants stop expanding after upgrade and must be re-created as named variants in the console. The v2.0.3 suffix-preset editor is replaced by the new variant editor.

### 简体中文

#### 新增

- **命名模型变体。** 模型变体现在拥有与基础模型名解耦的**自定义全名**——`gpt-5.5` 的一个变体可以直接叫 `gpt-image-2`(或任意名),并绑定一种强制行为,如画图工具、推理档位或 service tier。Provider 模型编辑页新增结构化变体编辑器;每个变体的行为会编译成一条按变体全名匹配的 rewrite 规则,变体在上游解析回其基础模型。
- **Transform 规则模板。** 规则编辑器新增一键 transform 模板,覆盖常见的客户端身份归一化(Aider、Cline、Continue、Cursor、OpenCode),并增强 transform 配置——支持多 path、显式 `from` / `with` 的文本替换。
- **用量页缓存 token 列。** 用量表格与图表现在在 input / output 之外展示 cache-write、cache-read 的 token 数,并在 file、SQLite、libSQL 后端补齐 rollup 与持久化支持。

#### 调整

- **变体配置格式(破坏性,无向后兼容)。** `variants_json` 改为存储绝对变体名——`{ "expose_base": true, "variants": ["gpt-image-2"] }` 或裸名数组——旧的后缀形式(`["-thinking"]` / `{ "suffixes": [...] }`)不再解析。升级后已有的后缀变体将停止展开,需在 console 中按命名变体重新创建。v2.0.3 的后缀预设编辑器由新的变体编辑器取代。
