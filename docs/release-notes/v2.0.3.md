## v2.0.3

> Adds several console and routing workflow improvements, moves provider process rules from the old `sanitize` kind to generic `transform` rules, and refreshes Claude Code compatibility for the current 2.1.185 client behavior.

### English

#### Added

- **Scoped model aliases.** Model aliases can now be scoped by provider instead of being only global, so the same public model name can map differently per upstream while exports, imports, snapshots, and migrations keep the scope intact.
- **Admin batch operations.** Console tables now support batch enable / disable / delete flows across providers, credentials, models, rule sets, routes, users, teams, user keys, and usage records, with matching persistence support for file, SQLite, and libSQL backends.
- **Model variant presets.** Provider model editing gained suffix presets that can auto-create the matching rewrite rules for common provider-specific model variants.
- **Generic transform rules.** The process layer replaces `sanitize` rules with `transform` rules that can target request or response bodies by path or regex, including streaming response restoration.

#### Fixed

- **Claude Code OAuth endpoints.** The Claude Code channel again uses the v1 OAuth authorize/token endpoints expected by the current upstream flow.
- **Claude Code 2.1.185 fingerprint.** The Claude Code request fingerprint now uses the 2.1.185 user agent/runtime headers and updated billing-header shape.

#### Changed

- **Built-in self-update repository.** The self-update repository is now compiled in instead of requiring a runtime repository setting.
- **Rules console cleanup.** The old sanitize template/editor UI was removed; transform rules are edited as JSON and summarized in the rule pipeline.

### 简体中文

#### 新增

- **按 provider 作用域的模型别名。** 模型别名现在可以按 provider 限定，不再只能全局生效；同一个公开模型名可在不同上游映射到不同目标，导入导出、snapshot 与迁移都会保留作用域。
- **管理端批量操作。** Console 表格现在支持批量启用 / 禁用 / 删除 providers、credentials、models、rule sets、routes、users、teams、user keys 和 usage records，并补齐 file、SQLite、libSQL 后端持久化支持。
- **模型变体预设。** Provider model 编辑页新增 suffix preset，可为常见 provider-specific 模型变体自动创建对应 rewrite rules。
- **通用 transform 规则。** process 层用 `transform` 取代旧 `sanitize`，可按 path 或 regex 改写请求 / 响应 body，并支持流式响应还原。

#### 修复

- **Claude Code OAuth endpoint。** Claude Code 渠道恢复使用当前上游流程需要的 v1 OAuth authorize/token endpoint。
- **Claude Code 2.1.185 指纹。** Claude Code 请求指纹更新到 2.1.185 的 User-Agent / runtime headers 和 billing header 形状。

#### 调整

- **内置 self-update 仓库。** self-update 仓库改为编译进二进制，不再需要运行时配置 repository。
- **规则 Console 清理。** 移除旧 sanitize template/editor UI；transform 规则改为 JSON 编辑，并在规则管线中显示摘要。
