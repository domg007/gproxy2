## v2.0.2

> Fixes the Claude Code channel against the current client behavior: auth-code login uses the claude.ai consent page, token exchange stays on api.anthropic.com with the required OAuth headers, session-cookie login normalizes pasted cookies, and the Claude Code billing checksum matches the 2.1.178 client.

### English

#### Fixed

- **Claude Code OAuth login.** The auth-code flow now uses `https://claude.ai/oauth/authorize`, exchanges tokens at `https://api.anthropic.com/v1/oauth/token`, and includes the `state` / `origin` / OAuth beta headers required by the token endpoint.
- **Claude Code cookie login.** Session-cookie credentials now accept the Console's `sessionKey=...` value, a full `Cookie:` header, or the bare `sk-ant-sid...` value, and avoid sending a duplicated `sessionKey=sessionKey=...` cookie during bootstrap / authorize.
- **Claude Code request fingerprint.** The channel now sends the current `claude-code/2.1.178` user agent, Stainless SDK `0.94.0` headers, and the 2.1.178 dynamic `cc_version` suffix used by the bundled client before computing `cch`.

#### Changed

- Removed a stale local Claude worktree gitlink from the repository tree.

### 简体中文

#### 修复

- **Claude Code OAuth 登录。** auth-code 流程现在使用 `https://claude.ai/oauth/authorize`，在 `https://api.anthropic.com/v1/oauth/token` 换 token，并带上 token endpoint 要求的 `state` / `origin` / OAuth beta headers。
- **Claude Code cookie 登录。** session-cookie 凭据现在接受 Console 里的 `sessionKey=...`、完整 `Cookie:` header，或裸的 `sk-ant-sid...` 值，并避免 bootstrap / authorize 时发出 `sessionKey=sessionKey=...` 这种重复 cookie。
- **Claude Code 请求指纹。** 渠道现在发送当前的 `claude-code/2.1.178` User-Agent、Stainless SDK `0.94.0` 头，并按 2.1.178 bundle 的动态 `cc_version` 后缀计算 `cch`。

#### 调整

- 从仓库树里移除了一个过期的本地 Claude worktree gitlink。
