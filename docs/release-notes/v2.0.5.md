## v2.0.5

> Restores the top-level (global) Claude cache breakpoint target dropped in the v2 rewrite, fixes the cache-breakpoint `index` to honor its documented signed semantics, and tightens deployment security guidance.

### English

#### Added

- **Top-level (global) cache breakpoint.** The `cache_breakpoint` rule regained the `top_level` target (v1's `global`), which stamps `cache_control` on the Claude request root to enable Anthropic's automatic prompt caching. It is selectable in the console rule editor, and the irrelevant block-index field is hidden for this target.

#### Fixed

- **Cache breakpoint `index` semantics.** The `index` field now follows its documented signed, 1-based rule — `>0` counts from the start, `<0` from the end, `0` is invalid, and an omitted index targets the last block. Previously it was treated as a raw 0-based index, which mis-targeted negative indices (everything but `-1` fell back to the last block) and was off by one for positive values.

#### Changed

- **Deployment guidance.** The Docker quickstart and README now require a 12-character admin password and flag serving the admin cookie without `Secure` over plain HTTP.

### 简体中文

#### 新增

- **顶层(全局)缓存断点。** `cache_breakpoint` 规则恢复了 `top_level` target(对应 v1 的 `global`),它把 `cache_control` 写在 Claude 请求根对象上,启用 Anthropic 的自动 prompt 缓存。可在 console 规则编辑器中选择;该 target 下会隐藏无意义的 block 索引字段。

#### 修复

- **缓存断点 `index` 语义。** `index` 字段现在遵循文档约定的有符号、1-based 规则——`>0` 从头数,`<0` 从尾数,`0` 无效,省略时指向最后一个 block。此前它被当成原始 0-based 索引,导致负数索引失效(除 `-1` 外都退回最后一块)、正数差一。

#### 调整

- **部署指引。** Docker 快速上手与 README 现在要求 12 位管理员密码,并提示在明文 HTTP 下使用未加 `Secure` 的管理 cookie 的风险。
